【题解】公园

2023-07-24 12 min read

题目

题目来源：第三届锡监杯数学挑战赛-T5

原版题面

  秋日的午后，带npy逛公园多是件美事啊……
  公园中有 nnn 个景点，景点间由 mmm 条道路连接，保证景点间可相互走通，每条道路连接两个不同的景点，不存在两条道路连接相同的两个景点。
   小S和npy决定打卡公园的每一个景点（即每个人都要到达所有的景点），他们分别从 S1S_1S1​、S2S_2S2​ 景点出发（ S1S_1S1​ 可能与 S2S_2S2​ 相同），最后回到各自的出发点。每 111 单位时间内，其中一人欣赏景点，另一人可通过一条道路到达相邻的景点。
  公园很大，他们怕自己逛不完，于是利用魔法，可以在 111 单位时间内交换小S和npy两人的位置。
  他们还有很多别的事要做，因而他们想在最短时间内逛完公园，最短时间为 kkk。
  小S通过敏锐的数学直觉发现，不论公园景点数与道路情况如何，都有 k<cn+6k<cn+6k<cn+6 成立。为了安排行程，请你帮助他求出 ccc 的最小值。

简化版题面

  给定一张 nnn 点 mmm 边的无向连通图（无重边、自环），有两个人A和B，出发点 S1S_1S1​, S2S_2S2​，每一步可以进行任意一步以下操作：
  1. A沿一条边走到相邻位置； 2. B沿一条边走到相邻位置； 3. 交换A和B的位置。
  要使A、B均遍历图上的每一个节点，且最后回到出发点。可以任意选择出发点和每一步操作，操作最小数量是 kkk。
  对于任意符合要求的图，都有 k<cn+6k<cn+6k<cn+6 成立，求 ccc 的最小值。

解析

本题思路为构造，即给出一种符合要求的遍历方法，使得最坏情况下操作数尽可能少。
我们不妨把所给的连通图简化它的为任意一棵生成树。
由于题中要求对于任意符合要求的图均成立，故上述简化对求解无影响。

Subtask 111

若不考虑交换操作，最直接想到的就是每个人各自遍历生成树。
一人遍历生成树并回到出发点，每条边都被经过 222 次，操作数 2n−22n-22n−2。
总操作数 4n−44n-44n−4，
此算法下 ccc 最小取值为 444。

Subtask 222

考虑交换操作，
若A、B两人分别位于X、Y两点，连续进行两次交换操作后，两人位置不变，但到达过对方所在位置
一个人遍历，而另一个人位于原来位置不动，
前一个人每到一个新的点就交换两次，遍历完成后，后一个人其实也已完成遍历
总操作数 k=2(n−1)+(2n−2)=4n−4k=2(n-1)+(2n-2)=4n-4k=2(n−1)+(2n−2)=4n−4，
与Subtask 111 中相同。

考虑进一步优化，
让两人出发点相同（可任意选），于是A、B两人“地位”相同，
在前一种方案基础上，每到一个新的点后只需交换一次而非两次，
随后相当于代替彼此，继续遍历，
总操作数 k=(n−1)+(2n−2)=3n−3k=(n-1)+(2n-2)=3n-3k=(n−1)+(2n−2)=3n−3，
此算法下 ccc 最小取值为 333。

Subtask 333

每人已到达过的点构成一个遍历集合，
操作1和2（行走操作）最好情况能使一个人到达新点，即对遍历集合的贡献最多为 111，
操作3（交换操作）最好情况能使两个人都到达新点，即对遍历集合的贡献最多为 222，
如此看来，交换操作似乎更加划算（毕竟是小S的魔法）

注意到上一种方案中，起始点位总有一个人，
因此交换操作只能使一个人到达新点，即对遍历集合的贡献只有 111，
贡献没拉满怎么行呢
而如果A、B两个人都位于对方未到达过的点，此时交换后双方都到达新点，贡献为 222！
于是想到，可以让两人在不同的区域遍历，随后交换。

比如说，可以任取一个度数至少为2的节点为根，
取根节点为出发点，
A、B都在根的不同子树内遍历，可保证A、B到达的区域对于彼此而言都是“新”的。
A、B各自进行行走操作，直到到达新的节点，
随后进行交换操作，继续在子树内遍历。
最后，可能会出现一人在遍历最后一棵子树，而另一人停在出发点的情况。
此情况下就用 Subtask 222 中的算法。
遍历操作数 2n−22n-22n−2，交换操作数 (n−1−d)/2+d=(n−1+d)/2(n-1-d)/2+d=(n-1+d)/2(n−1−d)/2+d=(n−1+d)/2
总操作数 k=2.5n+0.5d−2.5k=2.5n+0.5d-2.5k=2.5n+0.5d−2.5，
其中 ddd 表示两人通过行走操作到达的新节点数之差。
通俗理解 ddd 的含义，就是一个人停了，另一个人还差多少节点没遍历完。

取树的重心作为根节点，
若只有两棵子树，则d≤1d\le1d≤1；
若有三棵及以上子树，
子树按照其包含节点数从大到小排序，前两棵子树节点数分别为 a,ba, ba,b。
剩余子树总节点数为 ccc，
将 a,b,ca, b, ca,b,c从大到小排序，记为 x,y,zx, y, zx,y,z。
满足 x+y+z=n−1x+y+z=n-1x+y+z=n−1，且 z≤y≤x≤n/2z\le y\le x\le n/2z≤y≤x≤n/2。
d≤∣x−y−z∣=∣2x−n+1∣d\le |x-y-z|=|2x-n+1|d≤∣x−y−z∣=∣2x−n+1∣
若 x=n/2x=n/2x=n/2，则 d=1d=1d=1；
否则 d=n−1−2xd=n-1-2xd=n−1−2x，
因为 x⩾n/3x\geqslant n/3x⩾n/3， 所以 d≤n/3−1d\le n/3-1d≤n/3−1。
于是 k≤8n/3−3k\le 8n/3-3k≤8n/3−3。
此算法下 ccc 最小取值为 8/38/38/3。

附-辅助工具

以下提供原题构造的代码实现，可辅助验证。
输入格式：
第一行两个整数 n,mn, mn,m，含义如题，nnn 不超过 5∗1055*10^55∗105。
接下来 mmm 行，每行两个整数 x,yx, yx,y，表示 xxx 节点和 yyy 节点之间连边。
输出格式：
第一行三个整数 k,S1,S2k, S_1, S_2k,S1​,S2​，含义如题；
接下来 kkk 行，有以下三种格式：

  1. A xxx，表示A移动到 xxx 节点；
  2. B xxx，表示B移动到 xxx 节点；
  3. Swap，表示将A和B位置互换。

代码

#include <bits/stdc++.h>
using namespace std;
const int N = 5e5 + 50;
int n,m,num,root;
int sum_blue,sum_org;
int head[N],fa[N],sum[N],maxn[N];
int step_blue[N*4],step_org[N*4];
int p[N*4],q[N*4];
bool unvis_blue[N*4],unvis_org[N*4],exc;
struct node{
	int to,nx;
}edge[N*2];
struct zj{
	int to,tot;
}a[N];
int blue[N],org[N];
void addedge(int x,int y) {
	edge[++num].to=y;
	edge[num].nx=head[x];
	head[x]=num;
}
void findfa(int x) {
	if (fa[x]==x)  return;
	findfa(fa[x]);
	fa[x]=fa[fa[x]];
}
void work(int u,int fa) {
	int i,v;
	sum[u]=1;
	for (i=head[u];i;i=edge[i].nx) {
		v=edge[i].to;
		if (v==fa)  continue;
		work(v,u);
		sum[u]+=sum[v];
		maxn[u]=max(maxn[u],sum[v]);
	}
	maxn[u]=max(maxn[u],n-sum[u]);
}
bool cmp(zj x,zj y) {
	return x.tot>y.tot;
}
void dfs(int u,int fa,int *step,bool *unvis) {
	int i,v;
	step[++num]=u;
	unvis[num]=1;
	for (i=head[u];i;i=edge[i].nx) {
		v=edge[i].to;
		if (v==fa)  continue;
		dfs(v,u,step,unvis);
		step[++num]=u;
	}
}
int main() {
	int i,j,v,x,y;
	scanf("%d%d",&n,&m);
	for (i=1;i<=n;i++)
		fa[i]=i;
	for (i=1;i<=m;i++) {
		scanf("%d%d",&x,&y);
		findfa(x), findfa(y);
		if (fa[x]==fa[y])  continue;
		fa[fa[x]]=fa[y];
		addedge(x,y);
		addedge(y,x);
	}
	work(1,-1);
	for (i=1;i<=n;i++)
		if (maxn[i]*2<=n) { root=i; break; }
	for (i=1;i<=n;i++)
		sum[i]=0, maxn[i]=0;
	work(root,-1);
	num=0;
	for (i=head[root];i;i=edge[i].nx) {
		v=edge[i].to;
		a[++num].to=v;
		a[num].tot=sum[v];
	}
	sort(a+1,a+num+1,cmp);
	for (i=1;i<=num;i++)
		if (sum_blue<=sum_org)  blue[++blue[0]]=a[i].to, sum_blue+=a[i].tot;
		else  org[++org[0]]=a[i].to, sum_org+=a[i].tot;
	num=0;
	for (i=1;i<=blue[0];i++) {
		dfs(blue[i],root,step_blue,unvis_blue);
		step_blue[++num]=root;
	}
	num=0;
	for (i=1;i<=org[0];i++) {
		dfs(org[i],root,step_org,unvis_org);
		step_org[++num]=root;
	}
	num=0;
	for (i=1,j=1;step_blue[i]>0||step_org[j]>0;i++,j++) {
		while (!unvis_blue[i]&&step_blue[i]>0) {
			if (!exc)  p[++num]=1, q[num]=step_blue[i];
			else  p[++num]=2, q[num]=step_blue[i];
			i++;
		}
		while (!unvis_org[j]&&step_org[j]>0) {
			if (!exc)  p[++num]=2, q[num]=step_org[j];
 			else  p[++num]=1, q[num]=step_org[j];
			j++;
		}
		if (step_blue[i]==0&&step_org[j]==0)  break; 
		if (step_blue[i]>0) {
			if (!exc)  p[++num]=1, q[num]=step_blue[i];
			else  p[++num]=2, q[num]=step_blue[i];
		} 
		if (step_org[j]>0) {
			if (!exc)  p[++num]=2, q[num]=step_org[j];
			else  p[++num]=1, q[num]=step_org[j];
		} 
		p[++num]=3;
		exc^=1;
	}
	printf("%d %d\n",root,num);
	for (i=1;i<=num;i++) {
		if (p[i]==1)  printf("A %d\n",q[i]);
		if (p[i]==2)  printf("B %d\n",q[i]);
		if (p[i]==3)  printf("Swap\n");
	}
	return 0;
}
